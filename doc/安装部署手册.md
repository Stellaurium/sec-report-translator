# SEC Report Translator 安装部署手册

本文说明如何从 GitHub 获取项目、安装依赖、配置模型和 SEC 下载信息，并验证软件可以正常运行。

## 1. 环境要求

- Windows、macOS 或 Linux
- Python 3.10 或更高版本
- 可访问 SEC 网站的网络环境
- 一个 OpenAI-compatible Chat Completions API 服务

在 Windows PowerShell 中可以检查 Python：

```powershell
py -3.10 --version
```

如果系统中没有 `py` 启动器，也可以使用：

```powershell
python --version
```

## 2. 获取源码

```powershell
git clone https://github.com/your-org/sec-report-translator.git
cd sec-report-translator
```

如果是下载 zip 文件，解压后进入项目根目录即可。

## 3. 创建虚拟环境

Windows PowerShell：

```powershell
py -3.10 -m venv .venv
.\.venv\Scripts\python.exe -m pip install -U pip
```

macOS/Linux：

```bash
python3.10 -m venv .venv
./.venv/bin/python -m pip install -U pip
```

## 4. 安装项目

开发/本地使用推荐 editable 安装：

Windows：

```powershell
.\.venv\Scripts\python.exe -m pip install -e .
```

macOS/Linux：

```bash
./.venv/bin/python -m pip install -e .
```

安装后会生成命令行入口 `sec-translator`。

Windows 验证：

```powershell
.\.venv\Scripts\sec-translator.exe --help
```

macOS/Linux 验证：

```bash
./.venv/bin/sec-translator --help
```

## 5. 生成配置文件

Windows：

```powershell
.\.venv\Scripts\sec-translator.exe init-config -o sec-translator.toml
```

macOS/Linux：

```bash
./.venv/bin/sec-translator init-config -o sec-translator.toml
```

`sec-translator.toml` 是本地配置文件，通常包含 API key，不应该提交到 Git。项目 `.gitignore` 已默认忽略它。

## 6. 配置主模型

打开 `sec-translator.toml`，修改 `[model]`。

云端 API 示例：

```toml
[model]
base_url = "https://api.example.com/v1"
api_key = "YOUR_API_KEY"
model = "provider-model-name"
temperature = 0.1
top_p = 0.95
timeout_seconds = 300
max_retries = 3
context_window_tokens = 128000
max_output_tokens = 4096
```

本地 OpenAI-compatible 服务示例：

```toml
[model]
base_url = "http://localhost:1234/v1"
api_key = "local-key"
model = "local-model"
temperature = 0.1
top_p = 0.95
timeout_seconds = 300
max_retries = 1
context_window_tokens = 128000
max_output_tokens = 4096
```

注意：

- `base_url` 只写到 `/v1`，不要写 `/chat/completions`。
- API key 只保存在配置文件中。
- `context_window_tokens` 会影响 block 规划，应与模型真实上下文能力接近。
- `max_output_tokens` 应覆盖单个 batch 输出需求；当前推荐 batch 为单翻译单元，通常不需要过大。

## 7. 配置 fallback 模型

如果希望主模型失败时使用另一个模型兜底，启用 `[fallback_model]`：

```toml
[fallback_model]
enabled = true
base_url = "http://localhost:1234/v1"
api_key = "local-key"
model = "fallback-model"
temperature = 0.1
top_p = 0.95
timeout_seconds = 300
max_retries = 1
context_window_tokens = 128000
max_output_tokens = 4096
```

不需要 fallback 时：

```toml
[fallback_model]
enabled = false
```

## 8. 配置 SEC 下载身份

SEC 下载命令需要 `[sec]`：

```toml
[sec]
user_agent_name = "Your Name or Company"
user_agent_email = "your_email@example.com"
```

邮箱必须是合法格式。该信息会传给 `sec-edgar-downloader`，用于构造 SEC 请求身份。

## 9. 推荐翻译参数

通用推荐：

```toml
[cache]
enabled = true
cache_dir = ".sec-translator-cache"

[block]
enabled = true
max_input_ratio = 0.12
target_batch_units = 1
max_batch_chars = 80000
max_batches_per_block = 80
max_workers = 20
warmup_first = true
warmup_delay_seconds = 3.0
before_context_ratio = 0.10
block_body_ratio = 0.80
after_context_ratio = 0.10
```

如果 provider 并发能力强、网络稳定，可以把 `max_workers` 提高到 `50`。如果出现超时、429 或单个请求长时间卡住，优先降低到 `10` 到 `20`。

## 10. 验证安装

### 10.1 运行测试

Windows：

```powershell
.\.venv\Scripts\python.exe -m pytest
```

macOS/Linux：

```bash
./.venv/bin/python -m pytest
```

### 10.2 验证配置文件可读

```powershell
.\.venv\Scripts\sec-translator.exe init-config -o temp-config.toml
```

如果文件已存在，加 `--overwrite`。

### 10.3 验证下载

```powershell
.\.venv\Scripts\sec-translator.exe download --ticker AAPL --form 10-K --limit 1 -o downloads --config sec-translator.toml
```

成功后应看到类似目录：

```text
downloads/sec-edgar-filings/AAPL/10-K/<accession>/full-submission.txt
```

### 10.4 验证拆包

```powershell
.\.venv\Scripts\sec-translator.exe unpack downloads\sec-edgar-filings\AAPL\10-K\<accession>\full-submission.txt -o output\AAPL-10K
```

输出目录应包含 `manifest.json` 和多个 filing 文件。

### 10.5 验证翻译

先在拆包目录中找到主 HTML 文件，然后运行：

```powershell
.\.venv\Scripts\sec-translator.exe translate output\AAPL-10K\main.htm --config sec-translator.toml
```

成功后会生成：

```text
main_translated.htm
main_translation_report.json
main_translation_report.md
```

## 11. 部署建议

本项目是本地 CLI，不需要常驻服务。部署重点是：

- 固定 Python 版本。
- 使用虚拟环境隔离依赖。
- 不提交 `sec-translator.toml`。
- 将下载文件、拆包文件、翻译输出和缓存目录排除在 Git 外。
- 长文档翻译时保留磁盘缓存，避免中断后从头开始。

## 12. 升级项目

从 Git 更新后：

```powershell
git pull
.\.venv\Scripts\python.exe -m pip install -e .
.\.venv\Scripts\python.exe -m pytest
```

如果 prompt 或模型名变化，旧翻译缓存可能不会复用，这是正常行为。

## 13. 故障排查

### 命令找不到

确认已经安装：

```powershell
.\.venv\Scripts\python.exe -m pip install -e .
```

然后使用虚拟环境中的完整路径：

```powershell
.\.venv\Scripts\sec-translator.exe --help
```

### API 请求失败

检查：

- `base_url` 是否只到 `/v1`。
- `api_key` 是否正确。
- `model` 名称是否和 provider 一致。
- 网络是否可访问 provider。
- `timeout_seconds` 是否过短。

### 上下文长度错误

降低：

```toml
[block]
max_input_ratio = 0.08
max_batches_per_block = 40
```

或调低：

```toml
[model]
context_window_tokens = 128000
max_output_tokens = 4096
```

### 翻译输出中有少量原文

查看 Markdown 或 JSON 报告中的 `retained_units`。这通常表示模型输出未通过验证。可以启用 fallback 模型或降低并发后重试。
